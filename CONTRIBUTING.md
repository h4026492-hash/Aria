# Contributing to Aria

Thanks for your interest in contributing! Please follow these guidelines to keep the project healthy and easy to maintain.

## Code Style

- Follow TypeScript and React best practices.
- Use Prettier for formatting and ESLint for linting. Run `npm run lint` and `npm run format` before submitting PRs.

## Branch Naming

- Use `feature/<short-name>` for feature branches
- Use `bugfix/<short-name>` for bug fixes
- Use `sadtalker/<task-short-name>` for SadTalker-related work

## Pull Requests

- Open a PR against `main` with a clear description and a short list of changed files.
- Keep PRs small and focused. Include tests for new functionality where possible.

## Issues

- Use the issue template to file bugs or feature requests. Provide steps to reproduce and logs if relevant.

## Code of Conduct

Be respectful and constructive. This project follows the Contributor Covenant (https://www.contributor-covenant.org/). By participating, you agree to abide by this code of conduct.
